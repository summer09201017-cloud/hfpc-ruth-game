## ⚠️ 開發紀律（這次事故的教訓，務必遵守）

1. **做完即 commit + push**——未提交的源碼會被另一個 session 的 `git clean` 清掉（挪亞兩關就被清過一次，靠記憶重建）。
2. **多 session 同一工作樹**：一次只讓一個 session 動 git；切分支前先確認別人沒在用；危險操作（clean/reset/rm）前先 dry-run、用 stash。
3. **零美術檔關卡改完一定截圖驗收**（`.jsx` 不能 `node --check`）。
4. **文案一律牧者把關**再連到大廳/併 main（main 自動部署＝直接曝光）。
5. **本機 vite build 地雷**（agape250 機 Node 24 偶發無聲 exit 127）：先刪 `dist/` 再 build；HFPC 機與 Netlify(Node 20) 不受影響。
